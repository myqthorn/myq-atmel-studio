8 Register values are shown in hexadecimal format and should be ORed together.
9 For example, to set the Air Data Rate to 2Mhz and the RF output power to -6dB, you would write
10 the following to the RF_SETUP(0x06) register:
11 (NRF24L01_RF_DR_2MHZ | NRF24L01_RF_PWR_6DB)
12 
13 Bit locations are in decimal format and should be left shifted and ORed.
14 For example, to mask the RX_DR interrupt and mask the TX_DS interrupt, you would write
15 the following to the CONFIG(0x00) register:
16  ((1<<NRF24L01_MASK_RX_DR) | (1<<NRF24L01_MASK_TX_DS))
17 
18 to compare the bit value you would left shift and AND the value.
19 To check if the MASK_RX_DR bit has been set in the CONFIG register,

278 /*
280 //Enhanced ShockBurst Transmitting Payload
283 
284 1. Set the configuration bit PRIM_RX low.
285 
286 2. When the application MCU has data to transmit,
287 clock the address for the receiving node(TX_ADDR) and
288 payload data(TX_PLD) into nRF24L01+ through the SPI.
289 The width of TX-payload is counted from the
290 number of bytes written into the TX FIFO from the MCU.
291 TX_PLD must be written continuously while holding CSN low.
292 TX_ADDR doesn't have to be rewritten if it is unchanged from the last transmission.
293 If the PTX device shall receive acknowledge, configure data pipe 0 to receive the ACK packet.
294 TX_ADDR and RX_ADDR_P0 on the PTX device and RX_ADDR_Px on the PRX device must all be the same.
295 
296 3. Start transmission with a high (pulse) on CE. Minimum pulse width is 10uS.
297 
298 4. nRF24L01+ ShockBurst:
299  Radio is powered up.
300  16MHz internal clock is started.
301  RF packet is completed (see the packet description).
302  Data is transmitted.
303 
304 5. If auto Acknowledgment is activated (ENAA_P0 = 1), the radio goes into RX mode immediately,
305 unless the NO_ACK bit is set in the received packet. If a valid packet is received in the valid
306 acknowledgment time window, the transmission is considered a success. The TX_DS bit in the
307 STATUS register is set high and the payload is removed from TX FIFO.
308 If a valid ACK packet is not received in the specified time window, the payload is retransmitted
309 (if auto retransmit is enabled).
310 If the auto retransmit counter (ARC_CNT) exceeds the programmed maximum limit (ARC),
311 the MAX_RT bit in the STATUS register is set high. The payload in TX FIFO is NOT removed.
312 The IRQ pin is active(low) when MAX_RT or TX_DS is high.
313 To turn off the IRQ pin, reset the interrupt source by writing to the STATUS register.
314 If no ACK packet is received for a packet after the maximum number of retransmits,
315 no further packets can be transmitted before the MAX_RT interrupt is cleared.
316 The packet loss counter (PLOS_CNT) is incremented at each MAX_RT interrupt.
317 ARC_CNT counts the number of retransmits required to get a single packet through.
318 PLOS_CNT counts the number of packets that did not get through after the maximum number of retransmits.
319 
320 6. nRF24L01+ goes into standby-I mode if CE is low. Otherwise, the next payload in TX FIFO
321 is transmitted. If TX FIFO is empty and CE is still high, nRF24L01+ enters standby-II mode.
322 
323 7. If nRF24L01+ is in standby-II mode, it goes to standby-I mode immediately if CE is set low.
324 
326 //my description of transmission
328 
329 setup
330  set TX_ADDR = RX_ADDR_P0 = RX_ADDR_Px(on PRX device)
331  enable Auto ACK
332  make sure TX_DS IRQ is not masked if using interrupts
333 
334 sendPayload:
335  clears CE //not sure this is necessary
336  sets PRIM_RX low (transmitter mode)
337  sets CE to start transmission
338 
339 nRF24L01+ ShockBurst: (automagically taken care of)
340  Radio is powered up.
341  16MHz internal clock is started.
342  RF packet is completed (see the packet description).
343  Data is transmitted.
344  TX_DS bit is set indicating that data has been sent
345  IRQ is active if not masked.
346 
347 interrupts:
348  TX_DS:
349  data transmitted successfully
350  MAX_RT:
351  Maximum retries reached.
352  Flag set on STATUS register, transmission stops until cleared.
353 
354 interrupt handler
355  TX_DS IRQ:
356  clear CE to put back into standby-I mode
357  clear TX_DS interrupt by writing to STATUS register
358  MAX_RT IRQ:
359  *flush TX FIFO if data isn't critical
360  clear MAX_RT interrupt by writing to STATUS register
